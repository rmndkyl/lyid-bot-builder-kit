# Deep Analysis Techniques


Untuk project yang LAYAK BUILD (score >= 60), lakukan deep analysis:

**A. On-Chain Metrics** (jika live)
- `web_search` untuk: `"[project]" TVL volume DeFiLlama DeFi`
- Cek DeFiLlama, Dune Analytics, atau explorer untuk:
  - TVL (Total Value Locked) dan trend 7d/30d
  - Daily active users / unique wallets
  - Transaction volume
  - Growth rate (user acquisition)

**B. Tokenomics Analysis**
- `web_search` untuk: `"[project]" tokenomics supply distribution vesting`
- Extract:
  - Total supply vs circulating supply
  - Token distribution (% team, investors, community, airdrop)
  - Vesting schedule (cliff, unlock dates)
  - Inflation/deflation mechanism (burn, staking yield)

**C. Team & Background**
- `web_search` untuk: `"[project]" team founders background`
- Check:
  - Doxxed team? Previous projects? Track record?
  - GitHub activity (commits, contributors, open issues)
  - LinkedIn profiles (if available)
  - Previous project exits/scams

**D. Competitor Landscape**
- `web_search` untuk: `"[category]" crypto projects 2026`
- Identify:
  - Similar projects in same category
  - How does it differentiate?
  - Market share / mindshare comparison
  - Competitor airdrop history (what did they reward?)

**E. Airdrop Signal Analysis**
- `web_search` untuk: `"[project]" airdrop confirmed criteria snapshot`
- Check:
  - Has the project confirmed an airdrop?
  - What criteria have been mentioned? (volume, frequency, tasks)
  - Similar projects' airdrop history (rewards per wallet)
  - Snapshot timing (announced? estimated? past?)

**F. Automation-Specific Analysis**
- `web_search` untuk: `"[project]" bot automation github`
- Check:
  - What actions can be automated? (daily, swap, stake, social, referral)
  - Existing bots available? (GitHub search for project-bot)
  - Anti-bot measures detected? (captcha, rate limiting, fingerprint)
  - Cost per wallet (gas, proxy, captcha solving)

**G. ROI Estimation**
Estimate based on similar projects:
- Expected airdrop value per wallet (based on comparable projects)
- Cost to farm per wallet:
  - Gas: $[X] per tx × [Y] txs per day × [Z] days
  - Proxy: $[X] per month
  - Captcha: $[X] per 1000 solves
  - Time: [X] min setup per wallet
- Break-even wallet count: total_cost / expected_value_per_wallet
- Risk-adjusted return: expected_value × probability_of_airdrop

**H. Timing Analysis**
- Is it too early? (no farming mechanism, testnet only)
- Is it too late? (snapshot taken, TGE imminent, sybil filters active)
- Optimal farming window: [START] → [END]
- Recommended action: FARM NOW / WAIT / SKIP
